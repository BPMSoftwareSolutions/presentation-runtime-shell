Filters can either:

* ✅ make the system feel powerful
* ❌ or instantly make it feel like *more work / more software*

---

# The right answer: **Yes, but invisible-first**

She *can* use filters — but you should present them as:

> **“optional drill-down,” not required setup**

---

# How to position filters (important)

Do **NOT** say:

> “You can configure filters to generate reports.”

Say instead:

> “The report is already complete — filters just help you zoom in if needed.”

---

# Where filters appear in your slide

Add a **small, secondary section** — not the main focus.

# The 4 filters that actually matter

Keep it tight — don’t overload.

### 1. Status (most important)

```text
Used in Closing
Inactive / Ancillary
Superseded
Exceptions
```

👉 This is the one she’ll use most during audits.

---

### 2. Document Type

```text
Disclosure
Note
Title
Payoff
ID
Other
```

👉 Helps answer:
“Show me all disclosures used in this closing.”

---

### 3. Date / Timeline

```text
Intake date
Finalized date
Review actions
```

👉 Useful for audit timelines.

---

### 4. Reviewer / Actor (optional but powerful)

```text
Roni
Processor
System
```

👉 Answers: “Who made this decision?”

---

# What NOT to include (important)

Avoid:

* complex query builders
* advanced Boolean filters
* multi-step report configuration
* anything that looks like Excel

👉 That kills the simplicity instantly.

---

# How to demo filters (quick moment)

After showing the audit report, do **one simple interaction**:

```text
Click: Status → "Excluded / Inactive"
```

Then show:

```text
Escrow Worksheet
Addendum draft
Compliance Notice
```

Then say:

> “If an auditor asks what was excluded, you can isolate it instantly.”

That’s it. Don’t over-demo.

---

# The deeper product insight (this matters)

Filters are not the feature.

👉 **Clarity is the feature.**

Filters only exist because:

* everything is already classified
* every document already has a status
* every decision is already tracked

---

# The one-liner to use

> “You don’t need filters to make the report — you only use them when someone asks a specific question.”

---

# If you want to level this up further

The *really* powerful move (optional for later):

Instead of manual filters, add **quick audit questions**:

```text
[ Show excluded documents ]
[ Show final signed versions ]
[ Show all disclosures ]
[ Show decision history ]
```

👉 That feels smarter than filters.
